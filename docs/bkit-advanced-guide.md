## 1. 초보 vs 고수 한눈에 비교

```
┌───────────────────────────────────────────────────────────────────┐
│                    초보는 이렇게 써                                │
│                                                                   │
│   나 ──> /pdca plan ──> /pdca design ──> /pdca do ──> 끝!        │
│                                                                   │
│   문제점:                                                         │
│   ❌ 계획이 부실해서 나중에 다시 만듦                              │
│   ❌ 검증(analyze) 안 해서 설계랑 코드가 다름                     │
│   ❌ 커스텀 도구 활용 못 함                                       │
│   ❌ 매번 같은 명령어 수동 입력                                   │
└───────────────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────────────┐
│                    고수는 이렇게 써                                │
│                                                                   │
│   나 ──> /plan-plus (브레인스토밍)                                │
│      ──> /development-pipeline (9단계 체계)                       │
│      ──> 에이전트 팀 (여러 AI가 동시에!)                          │
│      ──> 자연어 트리거 ("검증해줘" 한마디로!)                     │
│      ──> 안전장치 훅 (위험한 명령 자동 차단)                      │
│      ──> Zero Script QA (로그로 테스트)                           │
│      ──> 커스텀 에이전트 (나만의 AI 도우미)                       │
│                                                                   │
│   장점:                                                           │
│   ✅ 브레인스토밍으로 계획 품질 UP                                │
│   ✅ 자동 검증으로 항상 90% 이상 품질                             │
│   ✅ 여러 AI가 동시에 작업해서 속도 UP                            │
│   ✅ 위험한 실수 자동 방지                                        │
└───────────────────────────────────────────────────────────────────┘
```

## 2. 고수 비기 #1: Plan Plus 브레인스토밍

### 초보의 계획 vs 고수의 계획

```
초보: /pdca plan 다크모드         ← 바로 계획서 작성 (생각 없이)
고수: /plan-plus 다크모드         ← 브레인스토밍 후 계획서 작성!
```

### Use Case Diagram

```
┌────────────────────────────────────────────────────────────┐
│                   Plan Plus 시스템                          │
│                                                            │
│  ┌─────────────────┐                                       │
│  │ 의도 발견        │  "진짜 뭘 원하는 거야?"               │
│  │ (Intent Discovery)│  → 숨은 요구사항 찾기                │
│  └────────┬────────┘                                       │
│           │                                                │
│  ┌────────▼────────┐                                       │
│  │ 대안 비교        │  "방법 A vs B vs C"                   │
│  │ (Alternatives)   │  → 장단점 비교표 작성                 │
│  └────────┬────────┘                                       │
│           │                                                │
│  ┌────────▼────────┐                                       │
│  │ YAGNI 검토       │  "이거 진짜 필요해?"                  │
│  │ (You Ain't Gonna │  → 불필요한 기능 제거                 │
│  │  Need It)        │                                      │
│  └────────┬────────┘                                       │
│           │                                                │
│  ┌────────▼────────┐                                       │
│  │ 최종 Plan 생성   │  더 정확한 계획서 탄생!               │
│  └─────────────────┘                                       │
│                                                            │
│  🧑‍💻 사용자                                                │
└────────────────────────────────────────────────────────────┘
```

### Sequence Diagram

```
 사용자              Claude              브레인스토밍 엔진
    │                   │                      │
    │ /plan-plus 다크모드│                      │
    │──────────────────>│                      │
    │                   │                      │
    │                   │── ① 의도 질문 ──────>│
    │                   │  "다크모드로 뭘       │
    │                   │   해결하고 싶어?"     │
    │                   │                      │
    │  "눈이 아파서..."  │                      │
    │──────────────────>│                      │
    │                   │                      │
    │                   │── ② 대안 탐색 ──────>│
    │                   │  "3가지 방법 비교"    │
    │                   │                      │
    │                   │<── 비교표 반환 ──────│
    │                   │  A: CSS 변수 (추천)  │
    │                   │  B: Tailwind dark:   │
    │                   │  C: 별도 CSS 파일    │
    │                   │                      │
    │  "A로 하자!"       │                      │
    │──────────────────>│                      │
    │                   │                      │
    │                   │── ③ YAGNI 검토 ─────>│
    │                   │  "시스템 테마 연동     │
    │                   │   진짜 필요해?"       │
    │                   │                      │
    │                   │<── 제거 항목 ────────│
    │                   │  "애니메이션 전환은   │
    │                   │   v2에서 하자"        │
    │                   │                      │
    │                   │── ④ Plan 문서 생성 ──│
    │  "완성된 계획서!"  │                      │
    │<──────────────────│                      │
```

**쉽게 말하면:** 고수는 바로 계획을 세우지 않아.
"진짜 뭘 원하는지" 먼저 파악하고, 여러 방법을 비교하고,
불필요한 건 빼고 나서야 계획을 세워!

## 3. 고수 비기 #2: 9단계 개발 파이프라인

### 초보는 3단계, 고수는 9단계

```
초보: Plan → Do → 끝                    (3단계, 대충)
고수: Schema → Convention → Mockup →     (9단계, 체계적!)
      API → Design System → UI →
      SEO/Security → Review → Deploy
```

### Use Case Diagram

```

│              9단계 개발 파이프라인                             │

### 각 단계 설명 (초등학생용)

| 단계 | 명령어 | 뭐하는 거야? | 비유 |
|------|--------|-------------|------|
| 1 | `/phase-1-schema` | 용어 사전 만들기 | 게임 시작 전 캐릭터 이름 정하기 |
| 2 | `/phase-2-convention` | 코딩 규칙 정하기 | "우리 반 규칙" 정하기 |
| 3 | `/phase-3-mockup` | 화면 그림 그리기 | 집 짓기 전 도면 그리기 |
| 4 | `/phase-4-api` | 서버 설계하기 | 식당 주방 메뉴판 만들기 |
| 5 | `/phase-5-design-system` | 부품 만들기 | 레고 블록 만들기 |
| 6 | `/phase-6-ui-integration` | 화면+서버 연결 | 레고 조립하기 |
| 7 | `/phase-7-seo-security` | 보안+검색 최적화 | 집에 자물쇠 달기 |
| 8 | `/phase-8-review` | 최종 검사 | 선생님 검사 받기 |
| 9 | `/phase-9-deployment` | 세상에 공개! | 가게 오픈! |

## 4. 고수 비기 #3: 에이전트 팀 운영 (CTO 모드)

## 5. 고수 비기 #4: 커스텀 에이전트/명령어 만들기

### 초보 vs 고수

```
초보: bkit 기본 도구만 사용
고수: 프로젝트 전용 도구를 직접 만듦!
