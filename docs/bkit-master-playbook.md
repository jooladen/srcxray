## 1. 왜 고수는 다르게 쓸까?

### 핵심 차이: "도구를 쓰는 순서"가 다르다

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   초보의 머릿속:                                             │
│   "일단 코드부터 짜자!" → 문제 생김 → 다시 짜기 → 반복...   │
│                                                             │
│   ┌────┐    ┌────┐    ┌────┐    ┌────┐                      │
│   │코드│───>│에러│───>│수정│───>│에러│───> 무한루프 😵       │
│   └────┘    └────┘    └────┘    └────┘                      │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   고수의 머릿속:                                             │
│   "먼저 생각하자!" → 설계 → 코드 → 검증 → 한 번에 완성!     │
│                                                             │
│   ┌────┐    ┌────┐    ┌────┐    ┌────┐    ┌────┐            │
│   │생각│───>│설계│───>│코드│───>│검증│───>│완성│ 😎          │
│   └────┘    └────┘    └────┘    └────┘    └────┘            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 초등학생 설명

```
포트폴리오 만들기는 마치 "자기소개 포스터" 만들기랑 같아!

1단계: 뭘 넣을지 생각해 (이름, 사진, 내가 한 것들)
2단계: 어떻게 배치할지 종이에 그려봐 (목업)
3단계: 컴퓨터로 진짜 만들어 (코딩)
4단계: 잘 만들어졌는지 확인해 (검증)
5단계: 인터넷에 올려! (배포)

포인트: 고수는 2단계(목업)를 절대 건너뛰지 않아!
        "먼저 그림 그리고 → 코딩" 이 순서가 핵심이야.
```

## 3. 시나리오 B: 팀으로 만드는 쇼핑몰

> 상황: "로그인 + 상품 목록 + 장바구니 + 결제가 있는 쇼핑몰!"

### Use Case Diagram — 쇼핑몰에 필요한 기능들

```
┌────────────────────────────────────────────────────────────────┐
│                Dynamic 레벨 쇼핑몰 프로젝트                      │
│                                                                │
│  🧑‍💻 나                                                        │
│    │                                                           │
│    │  ┌─────────── 사용자 기능 ──────────────┐                 │
│    ├──┤                                      │                 │
│    │  │  (UC-1) 회원가입/로그인               │                 │
│    │  │    → bkend-auth가 처리               │                 │
│    │  │                                      │                 │
│    │  │  (UC-2) 상품 목록 보기               │                 │
│    │  │    → bkend-data가 처리               │                 │
│    │  │                                      │                 │
│    │  │  (UC-3) 장바구니 담기/빼기            │                 │
│    │  │    → bkend-data가 처리               │                 │
│    │  │                                      │                 │
│    │  │  (UC-4) 주문/결제                    │                 │
│    │  │    → bkend-data + 외부결제API        │                 │
│    │  │                                      │                 │
│    │  │  (UC-5) 상품 이미지 업로드            │                 │
│    │  │    → bkend-storage가 처리            │                 │
│    │  └──────────────────────────────────────┘                 │
│    │                                                           │
│    │  ┌─────────── 관리자 기능 ──────────────┐                 │
│    └──┤                                      │                 │
│       │  (UC-6) 상품 등록/수정/삭제           │                 │
│       │    → RBAC 권한 필요 (admin만)         │                 │
│       │                                      │                 │

└────────────────────────────────────────────────────────────────┘
```

## 10. 마스터 체크리스트

### 프로젝트 시작 전

```
□ 레벨 결정했나? (Starter / Dynamic / Enterprise)
□ /plan-plus 로 브레인스토밍 했나?
□ 핵심 기능 MVP 범위 정했나?
□ 불필요한 기능 YAGNI 검토했나?
```

### 각 기능 개발 시

```
□ /pdca plan 으로 계획서 작성했나?
□ /pdca design 으로 설계서 작성했나?
□ /pdca do 로 설계 기반 구현했나?
□ /pdca analyze 로 검증했나? (90% 이상?)
□ 90% 미만이면 /pdca iterate 실행했나?
□ /pdca report 로 보고서 작성했나?
```

### 코드 품질

```
□ /build-check (typecheck + lint + build) 통과했나?
□ /code-review 실행했나?
□ 보안 취약점 없나? (/phase-7-seo-security)
□ 문서-코드 동기화 확인했나?
```

### 배포 전

```
□ /phase-8-review 최종 리뷰 했나?
□ /commit-push-pr 로 PR 생성했나?
□ /phase-9-deployment 배포 설정 확인했나?
```

### 고수 전용

```
□ 커스텀 에이전트 만들었나? (.claude/agents/)
□ 커스텀 명령어 만들었나? (.claude/commands/)
□ 안전장치 훅 설정했나? (.claude/settings.local.json)
□ 에이전트 팀 활성화 고려했나? (Enterprise 레벨)
```
