# Plan: inject-express (원클릭 주입 + 코드 미리보기)

## 배경

현재 DebugInjectorPanel은 **5단계 마법사**로 동작:
1. 분석 완료 → 2. 주입 중 → 3. 실행순서 → 4. 코드 복사 → 5. CSV

**문제점**:
- 매번 "다음" 버튼 4번 클릭해야 결과물에 도달
- 코드 복사는 되지만, **복사 전에 주입된 코드를 볼 수 없음**
- 가장 많이 쓰는 기능인데 UX가 무겁다

## 요구사항

### R1: 원클릭 주입 (Express Mode)
- 버튼 **1번만 누르면** 주입 완료 + 결과 화면 즉시 표시
- 기존 5단계 마법사는 "자세히 보기"로 접근 가능하게 유지

### R2: 코드 미리보기 (Injected Code Preview)
- 복사 전에 주입된 코드를 **시각적으로 확인** 가능
- 주입된 줄이 어디인지 **한눈에** 구분

## 설계 아이디어

### Express 결과 화면 (1페이지에 모든 것)

```
┌─────────────────────────────────────────────┐
│ ⚡ 로그 주입 완료!  15개 위치               │
│                                             │
│ ┌─ 📋 코드 미리보기 ─────────── [전체복사] ─┐│
│ │  1 │ function App() {                    ││
│ │  2 │▌console.log('[L:2][App][props]'...) ││ ← 녹색 하이라이트
│ │  3 │  const [count, setCount] = ...      ││
│ │  4 │▌console.log('[L:4][state] ...')     ││ ← 녹색 하이라이트
│ │  5 │  useEffect(() => {                  ││
│ │  6 │▌  console.log('[L:6][useEffect]')   ││ ← 녹색 하이라이트
│ │ ...│  (최대 300줄, 스크롤)                ││
│ └──────────────────────────────────────────┘│
│                                             │
│ [📋 전체 복사] [💾 원본 백업] [📊 CSV 다운로드] │
│                                             │
│ ▸ 실행순서 예측 (12단계)        [접기/펼치기] │
│ ▸ 주입 상세 내역                [접기/펼치기] │
│                                             │
│        [🔍 단계별로 자세히 보기]              │
└─────────────────────────────────────────────┘
```

### 핵심 아이디어

#### 1. Express 버튼
- Step 1의 "다음: Log 주입하기 →" 옆에 **"⚡ 바로 결과 보기"** 버튼 추가
- 이미 마운트 시 `Promise.all`로 모든 계산이 완료되어 있으므로 → 즉시 Express 화면으로 전환
- (또는) Step 1 자체를 Express 모드로 바꾸고, 기존 마법사를 "단계별 보기"로 분리

#### 2. 코드 미리보기
- `injectedSource`를 줄 단위로 split
- 각 줄에 `@@SRCXRAY-START` 마커가 있으면 → 다음 줄(console.log)에 녹색 배경 + 왼쪽 테두리
- `@@SRCXRAY-END` 마커까지의 영역을 하이라이트
- 마커 주석 자체는 렌더링에서 숨김 (시각적 노이즈 제거)
- 줄번호 표시 + 고정 높이(400px) + 스크롤
- 첫 번째 주입 포인트로 자동 스크롤

#### 3. 접이식 상세 영역
- **실행순서 예측**: 기존 Step 3 내용을 접기/펼치기(Collapsible)로 표시
- **주입 상세**: 기존 Step 1의 카테고리별 breakdown을 접기/펼치기로 표시
- **단계별 보기**: 기존 5단계 마법사로 진입하는 링크

없음 (기존 컴포넌트 내에서 구현)

### 변경하지 않을 파일

- `src/lib/ui-map-extractor.ts` — 변경 불필요

## 구현 순서

1. **CodePreview 서브컴포넌트** 구현
   - injectedSource를 줄 단위 렌더링
   - 마커 감지 → 하이라이트 처리
   - 줄번호 + 스크롤 + 자동 스크롤

2. **Collapsible 서브컴포넌트** 구현
   - 제목 + 접기/펼치기 토글

3. **Express 결과 화면** (step === 'express') 추가
   - CodePreview + 액션 버튼 + Collapsible 상세

4. **Step 1 UI 수정**
   - "⚡ 바로 결과 보기" 버튼 추가 (primary)
   - 기존 "다음: Log 주입하기" 버튼을 secondary 스타일로 변경

## 비즈니스 가치

| 지표 | 현재 | 목표 |
|------|------|------|
| 결과 도달 클릭 수 | 4회 | **1회** |
| 코드 확인 가능 여부 | 불가 (복사 후 확인) | **즉시 미리보기** |
| 기존 기능 유지 | - | 5단계 마법사 100% 유지 |

## 리스크

- 코드가 매우 길 경우 (1000줄+) 미리보기 렌더링 성능 → `virtualized list` 또는 높이 제한으로 해결
- 모바일에서 코드 미리보기 가독성 → 가로 스크롤 + 작은 폰트 사이즈
