# Plan: debug-injector-wizard — 5단계 위자드 UX + UI 맵 CSV 내보내기

**작성일**: 2026-02-23
**레벨**: Dynamic
**목표**: 주석주입기를 "쓰기 편한" 경험으로 개선. 스텝바이스텝으로 물개박수 5관왕 + 로그 대신 CSV로 화면별 버튼/핸들러 문서화

---

## 배경 & 문제 인식

현재 `DebugInjectorPanel`은 구현 완료된 상태. 그런데 실제로 써보면:

```
사용자 시각:
① 코드 업로드 → 분석 완료
② 화면에 "🔧 Log 주입하기" 버튼 보임 — "이게 뭐지? 누르면 어떻게 되지?"
③ 버튼 클릭 → 카운트 애니메이션 → 정보가 한꺼번에 와르르 쏟아짐
   - 통계 배지 + 복사 버튼 + 원복 버튼 + 실행순서 예측 + 사용 가이드
   "어... 이 중에 뭘 먼저 해야 하지?"
④ 막히는 순간 → 그냥 닫음
```

### 구체적 고통

| 문제 | 현상 |
|------|------|
| 정보 과부하 | 주입 후 모든 정보가 한 화면에 쏟아짐 |
| 다음 행동 불명확 | "복사 후 뭘 해야 해?" — 사용 가이드가 있지만 눈에 안 띔 |
| 완성감 없음 | 5단계를 완주했는지 알 수 없음 |
| 로그 의존 | 콘솔 열고 로그 확인 → 왕초보에게 여전히 장벽 |
| 문서화 불가 | "이 파일의 버튼이 다 어디 있는지" 정리할 방법 없음 |

## 새 기능 요구사항

### F-10: 5단계 위자드 (Stepper)

현재 패널을 "스텝 진행" 방식으로 전환. 한 번에 한 단계씩.

```
[Step 1] ──→ [Step 2] ──→ [Step 3] ──→ [Step 4] ──→ [Step 5]
  준비          주입          예측         복사&실행      완주!
   ●─────────────○────────────○────────────○────────────○
```

**Step 1: 준비 완료** (코드 로드 직후 자동)
```
┌─────────────────────────────────────────────┐
│  Step 1/5  ●○○○○                           │
│                                              │
│  🎉 코드 분석 완료!                         │
│  MyComponent · 12개 위치에 Log 넣을 수 있어요│
│                                              │
│           [다음: Log 주입하기 →]            │
└─────────────────────────────────────────────┘
```

**Step 2: Log 주입** (버튼 클릭 → 카운트업 WOW)
```
┌─────────────────────────────────────────────┐
│  Step 2/5  ●●○○○                           │
│                                              │
│  ████████████ 12/12개 ✅                   │
│  props:2  state:3  effect:2  fn:5          │
│                                              │
│           [다음: 실행 순서 예측 →]          │
└─────────────────────────────────────────────┘
```

**Step 3: 실행 순서 예측** (WOW-01 분리 표시)
```
┌─────────────────────────────────────────────┐
│  Step 3/5  ●●●○○                           │
│                                              │
│  🔮 이 코드가 이렇게 실행됩니다             │
│  1️⃣ MyComp 시작 → props 수신               │
│  2️⃣ count = 0 초기화                       │
│  3️⃣ 첫 render                              │
│  4️⃣ useEffect 실행 (mount)                 │
│                                              │
│           [다음: 코드 복사 →]               │
└─────────────────────────────────────────────┘
```

**Step 4: 복사 & 실행** (WOW-04 사용 가이드)
```
┌─────────────────────────────────────────────┐
│  Step 4/5  ●●●●○                           │
│                                              │
│  📋 복사 완료! 이렇게 하세요                │
│  Step 1. VSCode에서 원본 파일 열기          │
│  Step 2. Ctrl+A → Ctrl+V                   │
│  Step 3. npm run dev                        │
│  Step 4. F12 → Console 탭                  │
│  Step 5. 컴포넌트 사용 → [L:번호] 확인 🎉  │
│                                              │
│  [📋 코드 복사]   [다음: CSV 문서화 →]      │
└─────────────────────────────────────────────┘
```

**Step 5: 완주! + CSV 내보내기** (최종 WOW)
```
┌─────────────────────────────────────────────┐
│  Step 5/5  ●●●●●  🎊 완주!                │
│                                              │
│  🏆 분석 완료! 총 12개 위치 문서화됨        │
│                                              │
│  📊 버튼/이벤트 맵 다운로드                 │
│  화면별 모든 버튼·입력·핸들러를 엑셀로 정리 │
│  MyComponent.map.csv (5행)                  │
│                                              │
│  [⬇ CSV 다운로드]  [💾 원본 백업]  [처음으로]│
└─────────────────────────────────────────────┘
```

### F-11: UI 요소 맵 CSV 내보내기 ⭐ 핵심 신기능

로그 없이 "이 파일에 어떤 버튼/입력이 있고 어디서 실행되는지" 문서화.

**CSV 출력 형식** (엑셀에서 바로 열기 가능):

```csv
컴포넌트,요소타입,텍스트/name,줄번호,이벤트,핸들러함수,핸들러줄
MyComponent,button,로그인,42,onClick,handleLogin,55
MyComponent,button,취소,48,onClick,handleCancel,72
MyComponent,input,email,50,onChange,setEmail(state),15
MyComponent,input,password,51,onChange,setPassword(state),16
MyComponent,form,,38,onSubmit,handleSubmit,80
```

**파싱 전략**:
- AST에서 JSX 요소 순회 (`JSXOpeningElement`)
- 대상 태그: `button`, `input`, `a`, `select`, `textarea`, `form`
- 이벤트 속성 추출: `onClick`, `onChange`, `onSubmit`, `onKeyDown` 등
- 핸들러 이름 → `result.functions`와 매칭 → 줄번호 연결
- `setState` 패턴 감지 → "(state)" 표시

**파일명**: `{컴포넌트명}.ui-map.csv`

### WOW 순간 정의 (물개박수 5관왕)

| 순서 | Step | WOW 트리거 | 물개박수 이유 |
|------|------|-----------|------------|
| 1관왕 | Step 1 | 코드 로드 직후 → "12개 위치 발견!" 카드 등장 | "분석도 했는데 어디 넣을지도 알아?!" |
| 2관왕 | Step 2 | 카운트업 0→12 애니메이션 완료 | "숫자가 올라가는 게 살아있는 것 같아" |
| 3관왕 | Step 3 | "이런 순서로 실행됩니다" — React lifecycle 처음 이해 | "실행도 안 했는데 순서를 알아?!" |
| 4관왕 | Step 4 | 코드 복사 → "복사됨 ✅" 확인 + 단계별 가이드 | "뭘 해야 하는지 다 써줬네" |
| 5관왕 | Step 5 | CSV 다운로드 → 엑셀 열면 버튼 목록이 딱! | "이걸로 동료한테 설명하면 되겠다!" |

## 기술 제약

- 브라우저 전용 (`'use client'` 필수)
- 외부 API 없음 — 완전 오프라인
- CSV: BOM(`\uFEFF`) 추가 → 한글 엑셀 깨짐 방지
- 기존 `@babel/parser` AST 재활용 (추가 파싱 비용 없음)
- 새 패키지 없음
