## 브레인스토밍 배경

### 사용자 요구
- 왕초보 리액트 개발자 타깃 유지
- "우와, 이 기능 너무 좋다! 박수!" 반응을 유발하는 기능
- 4가지 아이디어 모두 구현 (우선순위 기반)

### React 왕초보가 자다가 일어나도 알아야 할 것
```
1. useState  → 값 바뀌면 화면 다시 그려진다
2. useEffect → 특정 시점에 코드를 실행한다
3. props     → 부모→자식으로 데이터가 흐른다
4. key       → map() 목록에서 각 항목을 구별한다
5. 이벤트핸들러 → 사용자 행동에 반응한다
6. 조건부렌더링 → 조건에 따라 다르게 보여준다
```

## 사용자 페르소나

**대상**: 왕초보 리액트 개발자

**기대 반응**

- "내 코드 이런 실수 하고 있었어?!" (A)
- "오 이래서 두 번 실행됐구나!" (C)
- "이것들만 알면 이 파일 이해 가능하겠다!" (D)

## 개선 목표 (WHAT)

### F-13: ⚠️ 위험 신호 감지기 (Danger Detector)
**"내 코드 어디가 조심해야 해요?"**

**감지 패턴 목록**:
| 코드 | 설명 | 심각도 |
|------|------|--------|
| `useEffect(fn)` deps 없음 + 내부 setState | 무한 렌더링 루프 가능 | 🔴 높음 |
| `.map()` return에 key prop 없음 | 목록 버그, 성능 저하 | 🟡 중간 |
| `useEffect(async () => ...)` | async 직접 사용, 클린업 불가 | 🟡 중간 |
| useState가 5개 이상 | 커스텀 훅 분리 권장 | 🔵 낮음 |
| 삼항연산자 3단계 중첩 | 가독성 저하 | 🔵 낮음 |

**UI 방향**:
- TldrCard에 위험 신호 뱃지 표시 (0개면 "✅ 위험 신호 없음!")
- 새 탭 "⚠️ 위험신호" 또는 기존 탭에 통합
- 각 신호: 이유 설명 + 해결 방법 + 해당 줄 번호

**기대 반응**: *"아 이래서 내 코드가 무한 새로고침 됐구나!!"*

---

### F-14: 📖 필수 개념 발견 카드 (Essential Concepts)
**"이론으로만 알던 개념, 실제 코드에서 발견!"**

**감지할 React 필수 개념**:
| 개념 | 코드 패턴 감지 | 필수도 |
|------|--------------|--------|
| 조건부 렌더링 | `? :`, `&&`, `\|\|` | ⭐⭐⭐ 자다깨도 |
| 목록 렌더링 | `.map(`, `.filter(` | ⭐⭐⭐ 자다깨도 |
| 이벤트 핸들러 | `onClick`, `onChange`, `onSubmit` | ⭐⭐⭐ 자다깨도 |
| 폼 처리 | `e.preventDefault()`, `FormEvent` | ⭐⭐ 중요 |
| 비동기 처리 | `async/await`, `try/catch` | ⭐⭐ 중요 |
| 스프레드 연산자 | `...prev`, `...items` | ⭐⭐ 중요 |
| 옵셔널 체이닝 | `?.`, `??` | ⭐ 알면 좋음 |
| 타입 단언 | `as Type`, `as unknown as` | ⭐ 알면 좋음 |

**카드 형식**:
```
📖 조건부 렌더링 — 자다깨도 OK ⭐⭐⭐
"isLoading이 true면 로딩, false면 내용을 보여줘"
🎯 비유: 신호등 — 빨간불이면 멈추고, 초록불이면 가요
📍 이 파일에서 발견: 3줄, 47줄, 89줄
```

**기대 반응**: *"오 이 파일에서 필수 개념이 다 나오네!"*

### F-15: 🕐 useEffect 실행 타임라인
**"이 코드 언제 실행되나요?"**

**3가지 실행 시점 시각화**:
```
🚀 마운트 시 한 번만  → useEffect(fn, [])
👀 [userId] 바뀔 때마다 → useEffect(fn, [userId])
🔁 매 렌더링마다       → useEffect(fn)
🧹 + 클린업 포함      → return () => cleanup()
```

**타임라인 UI**:
```
[페이지 열림] → [🚀 데이터 로드] → [userId 바뀜] → [👀 재조회] → [페이지 닫힘] → [🧹 정리]
```

**기대 반응**: *"오 이래서 두 번 실행됐구나!"*

---

### F-16: 🏆 마스터 체크리스트
**"이것들만 알면 이 파일 완전 이해!"**

**체크리스트 구성**:
- 파일에서 발견된 개념들을 난이도별 분류
- 각 항목: 개념명 + 필수도 + 간단 설명 + 학습 링크
- 클릭해서 "이미 알아요!" 체크 가능
- 전부 체크 시: "이 파일 완전 마스터!" 배너

**항목 예시**:
```
⭐⭐⭐ 자다깨도 알아야 함
  □ useState — 상태 관리 기초
  □ 조건부 렌더링 — A면 B, 아니면 C

⭐⭐ 중요
  □ useEffect — 사이드 이펙트 처리
  □ async/await — 비동기 처리

⭐ 알면 좋음
  □ 옵셔널 체이닝(?.) — 안전한 속성 접근
```

**기대 반응**: *"이것들만 알면 되겠다! 해볼 수 있겠는데?"*
