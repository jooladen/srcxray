# Plan: debug-injector — 토글형 console.log 자동 주입기

**작성일**: 2026-02-23 (v3 — 물개박수 UX 추가)
**레벨**: Dynamic
**목표**: 파일 로딩 후 원본은 절대 건드리지 않고, 버튼 토글로 주입/원복 전환 + 왕초보도 "와!" 하는 경험 제공

---

## 배경 & 문제 인식

```
소스 로드 → 어디에 log 넣을지 고민 → 수동 삽입 →
실행 → 로그 확인 → 제거 → 또 실행 → 반복...
                ↑ 이 전 과정이 너무 귀찮음
```

### 구체적 고통 포인트

| 고통 | 실제 상황 |
|------|----------|
| 수동 삽입 피로 | 200줄 파일에 log 20개 넣으려면 10분 소요 |
| 위치 파악 불가 | 로그 찍혀도 "몇 번째 줄에서 찍힌 건지" 모름 |
| 정신없는 콘솔 | 로그가 많으면 어느 게 어느 건지 구분 불가 |
| 제거가 더 귀찮음 | 넣는 것보다 빼는 게 더 어려움 |
| 원본 훼손 불안 | "혹시 원본을 망가뜨리면 어쩌지?" 걱정 |
| React 순서 모름 | 로그가 찍혀도 "왜 이 순서지?" 이해 안 됨 |

---

## 사용자 페르소나

**대상**: SrcXray 사용자 (왕초보 ~ 중급 개발자)
- 남의 코드를 이해하려고 SrcXray에 붙여넣은 상황
- "이 코드가 실제로 어떻게 흘러가는지" 직접 돌려보고 싶음
- React 실행 순서(lifecycle)를 모름
- 원본을 망가뜨릴까봐 조심스러움

---

## 핵심 UX 원칙

### "파일 로딩 = 원본 보존" 불변 규칙

```
파일 로드 → 분석 결과 표시 (원본 100% 유지)
                ↓
         [🔧 Log 주입] 버튼 클릭 (사용자 명시적 선택)
                ↓
         주입된 버전으로 화면 전환
         (원본은 메모리에 항상 보존)
                ↓
         [↩ 원복] 버튼 클릭
                ↓
         원본으로 즉시 복귀
```

**절대 원칙**: 로딩 시 자동 주입 없음. 사용자가 버튼을 눌러야만 주입.

---

## 마커 기반 주입 방식 (Method A)

```typescript
// @@SRCXRAY-START L:42
console.log('[L:42][MyComp][props]', { name, age });
// @@SRCXRAY-END
```

- 원복 시 마커 블록만 정규식으로 제거 → 100% 원복
- `originalSource`를 메모리에 보존하여 즉시 복귀 가능

## 기능 요구사항

### F-01: 토글 버튼 UI

```
원본 모드일 때:
┌────────────────────────────────────────────┐
│ 📄 원본 소스           [🔧 Log 주입하기]   │
└────────────────────────────────────────────┘

주입 모드일 때 (노란 테두리):
┌────────────────────────────────────────────┐
│ 🔧 Log 주입됨 ●        [↩ 원본으로 복귀]  │
│  ✅ 12개 위치 (props:2 state:3             │
│                effect:2 handler:5)         │
│  [코드 복사]  [원본 백업 다운로드]          │
└────────────────────────────────────────────┘
```

### F-02: Log 주입 엔진

| 주입 위치 | 로그 예시 |
|----------|----------|
| 컴포넌트 진입부 | `[L:12][MyComp][props] { name, onClick }` |
| useState 직후 | `[L:15][state] count = 0` |
| useMemo 선언 직후 | `[L:18][useMemo] filteredData =', filteredData, { items }` |
| useEffect 진입부 | `[L:20][useEffect] deps:[count] 시작', { count }` |
| 이벤트 핸들러 진입 | `[L:35][handleClick] 진입` |
| return JSX 직전 | `[L:50][MyComp] render` |

**삽입 알고리즘**: 줄 단위 배열 → **뒤에서부터 삽입** (앞 줄번호 유지)

### F-03: 상태 관리

```typescript
interface DebugInjectorState {
  mode: 'original' | 'injected';
  originalSource: string;   // 로딩 시점 — 절대 변경 안 됨
  injectedSource: string;   // 주입된 버전
  injectionCount: number;
  breakdown: { props: number; state: number; effect: number; handler: number; };
}
```

### F-04: 원클릭 원복

```typescript
// originalSource를 그냥 표시 — 재계산 없이 즉시 복원
setState({ mode: 'original' });
```

### F-05: 원본 백업 다운로드

```
다운로드: MyComponent.original.tsx
```

## 물개박수 기능 (WOW Features)

### WOW-01: 주입 전 "실행 순서 예측" 표시

왕초보가 가장 모르는 것 → **React가 어떤 순서로 실행되는가**.
주입 버튼 클릭 시, 코드를 실제 실행하기 전에 예측 순서를 먼저 보여줌.

```
🔮 이런 순서로 로그가 찍힐 거예요

  1️⃣  [L:12] MyComp 시작 → props 수신
  2️⃣  [L:15] count = 0 초기화
  3️⃣  [L:45] name = "" 초기화
  4️⃣  [L:50] 첫 번째 render
  5️⃣  [L:20] useEffect 실행 (mount — 처음 한 번만)
  ─────────────────────────────
  이후 setCount() 호출 시:
  6️⃣  [L:15] count 변경 → 재render
  7️⃣  [L:50] 두 번째 render
  8️⃣  [L:20] useEffect 재실행 (count 의존)
```

**왜 물개박수인가**: 실행도 안 했는데 "이 순서로 됩니다"를 보여주면
왕초보는 React lifecycle을 처음으로 직관적으로 이해함.

### WOW-02: 스마트 한국어 라벨

변수명을 분석해 로그 설명을 한국어로 자동 생성:

| 변수 패턴 | 자동 라벨 |
|----------|----------|
| `isOpen`, `isVisible`, `isLoading` | `켜짐/꺼짐 상태` |
| `count`, `index`, `num`, `total` | `숫자 카운터` |
| `email`, `password`, `token` | `인증 입력값` |
| `data`, `list`, `items`, `posts` | `데이터 목록` |
| `error`, `err` | `에러 상태` |
| `user`, `profile`, `me` | `사용자 정보` |

```
// 일반 log
console.log('[L:15][state] isLoading =', isLoading);

// 스마트 라벨 log
console.log('[L:15][state] 로딩중(켜짐/꺼짐) =', isLoading);
```

**왜 물개박수인가**: "어, 내 변수명을 읽고 설명해줬어!" — 마법처럼 느껴짐.

### WOW-03: 주입 완료 카운트업 애니메이션

[🔧 Log 주입하기] 클릭 후:

```
주입 중... ████████░░ 0개
주입 중... ████████░░ 4개
주입 중... ████████░░ 8개
✅ 완료!           12개 위치에 주입됨
```

진행 바 + 숫자가 올라가는 애니메이션 (0.5초).
완료 시 초록색으로 전환 + 작은 체크 이펙트.

**왜 물개박수인가**: 숫자가 올라가는 것만 봐도 기분 좋음. 작업이 "살아있음"처럼 보임.

### WOW-04: 사용 가이드 — "이렇게 쓰세요" 스텝

주입 완료 후, 코드 복사 버튼 아래에 단계별 안내:

```
📋 복사했으면 이렇게 하세요

  Step 1.  VSCode에서 원본 파일 열기
  Step 2.  Ctrl+A → Ctrl+V  (방금 복사한 코드 붙여넣기)
  Step 3.  앱 실행 (npm run dev)
  Step 4.  브라우저에서 F12 → Console 탭 열기
  Step 5.  컴포넌트 클릭하거나 사용해보기
           → [L:숫자] 로그가 찍히면 성공! 🎉
  Step 6.  로그 전체 복사 → 아래에 붙여넣기
           → 실행 타임라인 자동 분석 ✨
```

**왜 물개박수인가**: 왕초보는 "복사 후 어떻게 하지?" 막히는 순간 포기함.
이 가이드가 있으면 끝까지 성공 경험을 만들어줌.

### WOW-05: 로그 결과 붙여넣기 → 실행 타임라인 시각화 ⭐ 최고 WOW

실제로 실행한 콘솔 로그를 SrcXray에 다시 붙여넣으면:

```
[붙여넣기 영역]
[L:12][MyComp][props] { name: "홍길동" }
[L:15][state] count = 0
[L:50][MyComp] render
[L:20][useEffect] deps: count
[L:35][handleClick] 진입
[L:15][state] count = 1
[L:50][MyComp] render
```

SrcXray가 파싱해서 타임라인으로 변환:

```
⏱ 실행 타임라인 (실제 기록)
─────────────────────────────
① MyComp 시작 → props: name="홍길동"
② count=0 초기화
③ 첫 render
④ useEffect 실행 (mount)
── 버튼 클릭 ──────────────────
⑤ handleClick 실행
⑥ count: 0 → 1 (상태 변경!)
⑦ 재render (count 바뀌었으니까)
─────────────────────────────
총 7단계 실행 / 재render 1회
```

**왜 물개박수인가**: 완전한 루프 완성.
`분석 → 주입 → 실행 → 결과 분석`을 SrcXray 안에서 전부 할 수 있음.
왕초보가 "내 코드가 이렇게 돌아가는구나!"를 처음으로 눈으로 확인하는 순간.
