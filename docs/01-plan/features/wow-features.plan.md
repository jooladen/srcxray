# Plan: wow-features — 왕초보 개발자 물개박수 기능 모음

**생성일**: 2026-02-22
**상태**: Plan
**프로젝트**: SrcXray
**대상**: 프리랜서 왕초보 개발자 (코드 분석시 머리 아픈 사람)

---

## 1. 문제 정의 (Pain Point)

왕초보 프리랜서 개발자가 남의 TSX 소스를 분석할 때 겪는 고통:

| 고통 | 현재 상태 | 원하는 상태 |
|------|----------|------------|
| "이 파일이 뭘 하는 파일인지 모르겠다" | 직접 다 읽어야 함 | 한 줄 요약 자동 생성 |
| "데이터가 어디서 와서 어디로 가는지 모르겠다" | 머릿속으로 추적 | 화살표로 시각화 |
| "어디부터 읽어야 하는지 모르겠다" | 1번 줄부터 무작정 읽음 | 읽기 순서 자동 안내 |
| "이 훅이 왜 쓰이는지 모르겠다" | 공식 문서 찾아다님 | 초보자 말로 설명 |
| "복잡한 코드에 압도된다" | 포기 | 복잡도 시각화로 심리적 안정 |

---

## 2. 목표 (Objective)

> **"500줄 TSX 코드를 처음 보는 초보자도 10분 안에 '아 이런 코드구나!' 하고 고개 끄덕이게 만든다"**

### 성공 기준
- [ ] 초보자가 코드를 보고 30초 안에 "이 파일이 뭘 하는지" 파악 가능
- [ ] 읽기 순서 안내로 무작위 읽기 → 체계적 읽기로 전환
- [ ] 훅/패턴 설명을 듣고 "아 이게 그거구나!" 반응 유도
- [ ] 분석 결과를 복사해서 노션/메모에 바로 붙여넣기 가능

---

## 3. 물개박수 기능 목록 (Features)

### ⭐ F-01: 파일 한 줄 요약 카드 (File TL;DR)
**우선순위**: 최고 (P0)

분석 완료 시 맨 위에 등장하는 "이 파일 요약" 카드.
- 컴포넌트 수, 주요 상태, 렌더링 트리거를 바탕으로 자동 생성
- 예시: `"사용자 목록을 보여주는 컴포넌트. API에서 데이터를 받아 검색 필터링 후 표시"`

```
┌──────────────────────────────────────┐
│ 📋 이 파일 한 줄 요약                │
│                                      │
│  "사용자 목록 표시 + 검색 필터링    │
│   컴포넌트. 외부 API 연동."         │
│                                      │
│  컴포넌트 1개 · useState 3개 · 100줄 │
└──────────────────────────────────────┘
```

**구현 포인트**:
- `parser.ts` 분석 결과(import 패턴, 훅, props)로 텍스트 자동 조합
- AI 없이 템플릿 기반으로 생성

---

### ⭐ F-02: 훅 역할 자동 번역 (Hook Plain-Korean Explainer)
**우선순위**: 최고 (P0)

각 훅이 "왜" 쓰이는지 초보자 말로 자동 설명.

| 패턴 감지 | 생성되는 설명 |
|----------|-------------|
| `useState(false)` + 버튼 클릭 | "버튼을 누르면 켜지고/꺼지는 스위치입니다" |
| `useState([])` + useEffect fetch | "서버에서 목록을 받아와 저장하는 바구니입니다" |
| `useEffect(fn, [id])` | "id가 바뀔 때마다 자동으로 실행되는 자동화입니다" |
| `useCallback` | "함수를 매번 새로 만들지 않게 캐시해 성능을 올립니다" |

**구현 포인트**:
- 훅 이름 + 초기값 타입 + deps 패턴으로 규칙 기반 설명 생성
- `lib/hook-explainer.ts` 분리 구현

---

### ⭐ F-03: 데이터 흐름 화살표 (Data Flow Visualizer)
**우선순위**: 높음 (P1)

props가 어디서 들어와서 어떻게 JSX에 쓰이는지 시각적으로 표시.

```
[Props: userId] ──→ [useState: user] ──→ [useEffect: fetchUser] ──→ [JSX: <UserCard>]
```

SVG/CSS로 브라우저에서 렌더링. 클릭하면 해당 줄 번호로 이동.

**구현 포인트**:
- parser.ts에서 props→state→JSX 연결 고리 추출
- `FlowDiagram.tsx` 컴포넌트 신규 구현

---

### ⭐ F-04: 복잡도 히트맵 (Complexity Heatmap)
**우선순위**: 높음 (P1)

코드 전체를 라인별 복잡도로 색상 표시. 빨간 부분 = "여기가 핵심/어려운 부분".

```
  1  | import React...              ← 초록 (단순)
 23  | const [data, setData]...     ← 초록
 45  | useEffect(async () => {      ← 노랑 (중간)
 67  |   const res = await fetch... ← 빨강 (복잡)
```

**구현 포인트**:
- 줄별 중첩 깊이 + 토큰 수 + 훅 여부로 복잡도 점수 계산
- `CodeHeatmap.tsx` 컴포넌트 + 파싱 결과 연동

---

### F-05: 분석 결과 복사 버튼 (Copy-to-Clipboard Report)
**우선순위**: 중간 (P2)

분석 결과를 마크다운 텍스트로 변환해서 한 번에 복사.
노션, 슬랙, 메모장에 바로 붙여넣기 가능.

```markdown
# UserList.tsx 분석 결과
- 컴포넌트: UserList (100줄)
- import: react, @/lib/api
- 상태: users[], loading, search
- 역할: 사용자 목록 표시 + 검색 필터링
```

---

### F-06: 읽기 시간 실측 타이머 (Reading Timer)
**우선순위**: 낮음 (P3)

분석 시작 버튼 누를 때 타이머 시작. 각 탭을 보는 시간 측정.
마지막에 "당신은 X분 X초 만에 분석했습니다!" 표시 → 성취감 유발.

- Next.js 16 + Tailwind CSS 4 유지
- 신규 의존성 최소화 (SVG는 순수 CSS/inline으로)
