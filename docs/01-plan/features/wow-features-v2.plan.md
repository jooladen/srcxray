# Plan: wow-features-v2 — 왕초보 물개박수 기능 2차

**작성일**: 2026-02-22
**레벨**: Dynamic
**목표**: 왕초보 개발자가 "우와!" 하고 물개박수 칠 수 있는 UX 개선

---

## 배경 & 문제 인식

현재 SrcXray는 기능은 있지만 **왕초보가 "와, 이거 대박이다!" 하기엔 아쉬운 부분**이 있음.

### 현재 문제점

| 문제 | 영향 |
|------|------|
| 빈 화면에서 "뭘 붙여넣어야 하지?" 막힘 | 첫 인상 실패 |
| 학습가이드 진행 표시줄이 안 움직임 (w-0 하드코딩) | 인터랙션 없음 |
| 통계 숫자가 그냥 딱 나타남 | 감동 없음 |
| 훅 설명(hook-explainer)이 숨어있음 | 핵심 기능 미노출 |
| "이 코드 배워서 뭐 만드나요?" 동기 없음 | 학습 의욕 저하 |
| 복잡도 히트맵 - 왜 복잡한지 모름 | 실용성 부족 |

---

## 사용자 페르소나

**대상**: 왕초보 개발자
- 코드를 처음 접하는 단계
- "이 코드가 뭔 소리인지" 모르는 상태
- 작은 성공 경험에 크게 감동함
- 이해했을 때 공유하고 싶어함

## 개선 목표 (WHAT)

### F-07: 샘플 코드 즉시 분석 버튼
**"뭘 붙여넣어야 하지?"를 없애자**

- 빈 화면에 샘플 버튼 3개 표시
  - 🔐 로그인 폼 (useState + form)
  - 📋 할일 목록 (useEffect + map)
  - 🛒 상품 카드 (props + conditional)
- 클릭 한 번으로 분석 시작 → 즉시 "우와" 경험
- 샘플은 앱 내 embedded (외부 API 없음)

**기대 반응**: "오 클릭 하나로 바로 분석되네!"

---

### F-08: LearningGuide 체크박스 인터랙션
**"내가 이해했다!"를 표시하는 기쁨**

- 각 섹션에 체크박스 추가
- 체크하면: 진행 표시줄 채워짐 + 초록색 처리
- 100% 완료 시: 🎉 축하 배너 + 성취감 메시지
  - "와! 이 파일 완전히 파악했어요! 개발자 맞죠?"
- 체크 상태는 분석 결과와 함께 유지

**기대 반응**: "진짜 내가 이해한 부분 체크하니까 뿌듯하다!"

---

### F-09: 통계 카운트업 애니메이션
**숫자가 올라가면 왜 이렇게 기분이 좋지?**

- 분석 결과 로드 시 📏줄 / 🧩컴포넌트 / ⚡훅 / 📦Import
- 0에서 목표값까지 카운트업 애니메이션 (0.6초)
- 큰 숫자(100줄+)일수록 빠르게 올라가는 느낌
- `requestAnimationFrame` 기반, 부드러운 easing

**기대 반응**: "숫자가 올라가는 거 보는 게 왜 이렇게 재밌지?"

---

### F-10: 훅 설명 전용 "훅 번역" 탭
**훅을 비유로 설명하는 "우와 그런 뜻이었어?" 카드**

- 현재 hook-explainer.ts가 있지만 노출이 거의 없음
- 새 탭 "⚡ 훅 번역" 추가 (또는 기존 탭 개선)
- 각 훅을 카드로 표시:
  ```
  ⚡ useState(isOpen, setIsOpen)
  💡 켜고 끄는 스위치(토글) 상태입니다
  📌 패턴: 토글/불리언
  🎯 비유: 전등 스위치 — 누를 때마다 ON/OFF
  ```
- 비유(analogy) 추가 (현재 없음) → 왕초보 친화적
- Hook별 색상 카드 UI

**기대 반응**: "아 useState가 스위치였구나! 이제 이해됨!"

---

### F-11: "이 패턴으로 뭐 만들 수 있나요?" 카드
**동기부여 — 배워서 뭔가 만들 수 있다는 설렘**

- TldrCard 아래 또는 내부에 "📌 이 패턴 마스터하면 →" 섹션
- 감지된 패턴 기반 동기부여 메시지:
  - form + useState → "🛒 쇼핑몰 로그인 폼 만들 수 있어요!"
  - list + useEffect → "📰 뉴스 피드, 할일 목록 만들 수 있어요!"
  - auth → "🔐 회원가입 시스템 만들 수 있어요!"
  - search + filter → "🔍 검색 기능 있는 앱 만들 수 있어요!"
- 여러 패턴 = 여러 가능성 나열

**기대 반응**: "이거 배우면 쇼핑몰 만들 수 있다고?! 더 열심히 봐야겠다"

---

### F-12: 복잡도 히트맵 "왜 복잡한가요?" 툴팁
**빨간 줄을 보고 "왜 빨간지" 알아야 실용적**

- 복잡도 높은 줄에 hover 시 "복잡한 이유" 툴팁
- 예시:
  - 삼항연산자 2개 → "❓ 조건이 겹겹이 쌓여있어요"
  - 옵셔널체이닝 → "?. 가 많아서 조심해서 읽어야 해요"
  - 콜백 중첩 → "함수 안에 함수가 있어요"
- complexity.ts에 이유 코드(reason) 추가
- 왕초보가 "아 이래서 빨간 거구나" 이해 가능

**기대 반응**: "빨간 줄 보고 그냥 무서웠는데, 이유를 알았으니 한번 봐볼게요"
