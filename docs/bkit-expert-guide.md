# bkit 전문가 가이드 — UML로 배우기

> 초등학생도 따라할 수 있는 bkit PDCA 완전 정복 가이드

---

## 1. bkit이 뭐야?

bkit은 **"계획 → 설계 → 구현 → 검증"** 을 반복하면서 코드를 만드는 도우미야.
이걸 **PDCA 사이클**이라고 불러:

```
  Plan(계획) → Design(설계) → Do(구현) → Check/Act(검증/개선)
      ↑                                        |
      └────────── 90% 미만이면 다시! ──────────┘
```

**쉽게 말하면:**

| 명령어 | 역할 |
|--------|------|
| `/pdca plan` | 뭘 만들지 **계획표** 쓰기 |
| `/pdca design` | 어떻게 만들지 **설계도** 그리기 |
| `/pdca do` | 진짜 **코드** 짜기 |
| `/pdca analyze` | 설계대로 잘 만들었는지 **검사**하기 |
| `/pdca iterate` | 부족한 부분 **자동으로 고치기** |
| `/pdca report` | 완성 **보고서** 쓰기 |

## 4. 에이전트 Use Case Diagram — "AI 도우미들"

bkit에는 전문 AI 도우미(에이전트)들이 있어. 각각 맡은 일이 달라:

```
                        ┌─────────────────────┐
                        │     사용자 (나)      │
                        └──────────┬──────────┘
                                   │
            ┌──────────────────────┼──────────────────────┐
            │                      │                      │
            ▼                      ▼                      ▼
   ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
   │  gap-detector  │   │ code-analyzer  │   │pdca-iterator   │
   │ "검사관"        │   │ "품질 감독관"   │   │ "수리공"        │
   │                │   │                │   │                │
   │ 설계서와 코드가 │   │ 코드 품질이    │   │ 부족한 부분을   │
   │ 일치하는지 검사 │   │ 좋은지 분석    │   │ 자동으로 수정   │
   └────────────────┘   └────────────────┘   └────────────────┘
            │                      │                      │
            ▼                      ▼                      ▼
   ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
   │report-generator│   │design-validator│   │ starter-guide  │
   │ "기록원"        │   │ "설계 검토관"   │   │ "초보자 가이드" │
   │                │   │                │   │                │
   │ 완료 보고서를   │   │ 설계 문서에    │   │ 처음 시작하는   │
   │ 자동 작성      │   │ 빠진 게 없는지 │   │ 사람 도와주기   │
   └────────────────┘   └────────────────┘   └────────────────┘
```

### 에이전트 자동 트리거 키워드

에이전트는 특정 키워드를 말하면 자동으로 호출돼:

| 키워드 | 호출되는 에이전트 | 하는 일 |
|--------|-------------------|---------|
| "검증해줘", "verify" | gap-detector | 설계-코드 일치 검사 |
| "개선해줘", "improve" | pdca-iterator | 자동 반복 개선 |
| "분석해줘", "analyze" | code-analyzer | 코드 품질 분석 |
| "보고서", "report" | report-generator | 완료 보고서 생성 |
| "도움", "help" | starter-guide | 초보자 안내 |

## 6. 실전 치트시트

```
┌─────────────────────────────────────────────────┐
│           bkit 전문가 6단계 공식                  │
│                                                 │
│  STEP 1: /pdca plan 기능이름                     │
│          → "이런 거 만들 거야" 계획서 생성        │
│                                                 │
│  STEP 2: /pdca design 기능이름                   │
│          → "이렇게 만들 거야" 설계서 생성         │
│                                                 │
│  STEP 3: /pdca do 기능이름                       │
│          → 설계서 보고 코드 작성                  │
│                                                 │
│  STEP 4: /pdca analyze 기능이름                  │
│          → 설계 vs 코드 비교 (목표: 90% 이상)    │
│                                                 │
│  STEP 5: (90% 미만이면) /pdca iterate 기능이름   │
│          → 자동으로 부족한 부분 수정              │
│                                                 │
│  STEP 6: /pdca report 기능이름                   │
│          → 완료 보고서 자동 생성                  │
│                                                 │
│  💡 언제든 /pdca status 로 진행 상황 확인!       │
│  💡 다음 뭐 할지 모르면 /pdca next !             │
└─────────────────────────────────────────────────┘
```

## 7. 빠른 참조표

| 뭘 하고 싶어? | 뭘 입력해? | 어떤 일이 일어나? |
|---|---|---|
| 새 기능 시작 | `/pdca plan 기능이름` | 계획 문서 자동 생성 |
| 설계서 만들기 | `/pdca design 기능이름` | 기술 설계서 자동 생성 |
| 코드 짜기 | `/pdca do 기능이름` | 설계 기반 구현 |
| 잘 만들었나 확인 | `/pdca analyze 기능이름` | 설계-코드 일치율 측정 |
| 자동 수정 | `/pdca iterate 기능이름` | 90% 넘을 때까지 반복 수정 |
| 보고서 | `/pdca report 기능이름` | 완료 보고서 생성 |
| 지금 어디까지 했지? | `/pdca status` | 현재 진행 상황 표시 |
| 다음 뭐 해? | `/pdca next` | 다음 단계 자동 안내 |

## 8. docs/ 폴더 구조

PDCA를 실행하면 아래처럼 문서가 자동 생성돼:

```
docs/
├── 01-plan/           ← /pdca plan 결과물
│   └── features/
│       └── 기능이름.plan.md
├── 02-design/         ← /pdca design 결과물
│   └── features/
│       └── 기능이름.design.md
├── 03-analysis/       ← /pdca analyze 결과물
│   └── 기능이름.analysis.md
├── 04-report/         ← /pdca report 결과물
│   └── features/
│       └── 기능이름.report.md
└── .pdca-status.json  ← 현재 진행 상황 추적
```

> **기억할 것: 항상 Plan → Design → Do → Analyze 순서를 지키면 전문가!**

> AI는 완벽하지 않습니다. 중요한 결정은 항상 직접 확인하세요.
